# Standard Contractual Clauses Addendum - GitHub Transfer

Generated on: 2026-04-09
Tool: GitHub
Residency Risk: USA

## Purpose
This addendum documents the legal safeguard approach for using GitHub where personal data or operational metadata may be transferred outside the EU/EEA.

## Transfer Context
- Service Provider: GitHub
- Transfer Region: United States of America
- Data Context: source code metadata, user account identifiers, access logs, issue tracker content, and operational collaboration records

## SCC Safeguard Position
- Standard Contractual Clauses (SCC) are required for this transfer setup.
- GitHub use must be reviewed together with the applicable DPA and transfer mechanism documentation.
- Only the minimum necessary personal data should be processed within the tool.

## Internal Control Measures
- Restrict personal data stored in repositories, issues, and tickets.
- Use role-based access and least-privilege repository permissions.
- Separate HR or sensitive personal records from engineering systems.
- Maintain documented review of GitHub transfer safeguards in the Ops Pillar.
